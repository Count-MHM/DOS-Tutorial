Dos Tutorial

Hello World

In this chapter, we are going to do what all other programming tutorials do: printing Hello World!

In this series, I'm going to assume that you know a bit of C/C++ and x86-16 assembly, and know how to use Makefiles.

So first make a folder. And open the folder in VS Code.

Open Explorer in VS Code (via icon on top left or Ctrl+Alt+L).

Right click and make a file called main.cpp:

// main.cpp:
#include <stdio.h>
int main()
{
    printf("Hello World!\n");
    return 0;
}

Make another file called Makefile:

# Makefile
run : test.exe
    dosbox test.exe

test.exe : main.obj
    wlink system dos option dosseg option quiet file { $< } name $@

main.obj : main.cpp
    wpp -0 -bt=dos -ms -s -wx -zq -fo="$@" "$<"

clean : .SYMBOLIC
    -del test.exe main.obj main.err

From the top, select Terminal/New Terminal or press Ctrl+Shift+` to open a terminal.

You can use wmake run to build and run the program and wmake clean to clean anything that wmake run made

VS Code doesn't know that we are using Watcom, to fix that we have to create a folder called .vscode (only setting for VS Code should be placed inside .vscode) and a file called c_cpp_properties.json inside:

Explanation

main.obj : main.cpp
    wpp -0 -bt=dos -ms -s -wx -zq -fo="$@" "$<"
  • wpp is the Open Watcom C++ x86 16-bit Optimizing Compiler
  • -0 : Only use 8086 instruction set
  • -bt=dos : Operating system is dos
  • -ms : Small memory model
  • -s : Remove stack overflow checks
  • -wx : Set warning level to maximum setting
  • -zq : Operate quietly (display only error messages)
  • -fo="$@" : Output file is $@ ($@=main.obj in this case)
  • "$<" : Input file is $< ($<=main.cpp in this case)
test.exe : main.obj
    wlink system dos option dosseg option quiet file { $< } name $@
  • wlink is the Open Watcom Linker
  • system dos : Target system is DOS
  • option dosseg : The DOSSEG option tells the Open Watcom Linker to order segments in a special way.
  • option quiet : Don't print anything other than errors
  • file { $< } : Input files ($<=main.obj in this case)
  • name $@ : Output file name $@ ($@=test.exe in this case)
clean : .SYMBOLIC
    -del test.exe main.obj main.err
  • Deletes files test.exe main.obj main.err
run : test.exe
    dosbox test.exe
  • Runs test.exe with DOSBox
